## Detailed Category Reviews

### 1. Architecture & Design -- 9/10

**Strengths:**
- **Clean modular separation.** Each pipeline stage maps to one Python module: `ladder.py`, `mapping.py`, `alleles.py`, `calling.py`, `consensus.py`, `classify.py`. This directly reflects the 7-stage scientific pipeline, making the code structure intuitive for domain scientists.
- **Single Responsibility Principle (SRP)** is well-applied. `tools.py` handles all subprocess execution. `config.py` owns data loading and repeat dictionary management. `cli.py` is purely CLI orchestration with lazy imports.
- **Dependency direction is clean:** modules depend inward on `config.py` and `tools.py`, never on each other's internals. No circular imports.
- **Lazy CLI imports** (`from open_pacmuci.alleles import ...` inside Click functions) keep CLI startup fast and isolate module loading.
- **Dataclass-based configuration** (`RepeatDictionary`) provides a typed, immutable interface for pipeline configuration.

**Weaknesses:**
- Minor: the `calling.py` module handles both Clair3 invocation and VCF parsing/filtering -- these could be separated for even cleaner SRP, though the current grouping is defensible given the domain coupling.
- The `same_length` / `homozygous` allele disambiguation logic in `calling.py` (`disambiguate_same_length_alleles`) is somewhat complex (lines 318-423) and could benefit from being its own module or at least a class.

**Files reviewed:** `src/open_pacmuci/*.py` (all 10 modules), `cli.py`

### 12. Performance & Algorithm Design -- 9/10

**Strengths:**
- **O(1) classification lookup:** Pre-computed `seq_to_id` and `mutated_sequences` dictionaries enable constant-time exact match classification (`config.py:77-94`).
- **Streaming pipeline:** `minimap2 | samtools sort` piped via `subprocess.Popen` avoids holding full SAM in memory (`mapping.py:72-148`).
- **Single-row Levenshtein DP:** Edit distance uses O(n) space instead of O(nm) (`classify.py:21-46`).
- **Bounded probe search:** `max_indel_probe = 30` caps the search space for indel detection, avoiding exhaustive enumeration (`classify.py:355`).
- **Bidirectional classification:** When forward pass stalls on a large mutation, backward anchoring from 3' end recovers downstream repeats (`classify.py:237-317`).
- **Indel-valley splitting:** Novel algorithm for resolving close alleles merged in read-count space (`alleles.py:206-293`).
- **Intermediate file cleanup:** Temporary SAM and FASTQ files removed after use (`calling.py:132-133`).

**Weaknesses:**
- Edit distance fallback iterates over all repeat types linearly (`classify.py:181-185`). For the ~50 known types this is fine, but would not scale to larger repeat catalogs without indexing.
- No parallelism within the pipeline (allele 1 and allele 2 processed sequentially). Could use `concurrent.futures` for independent allele processing.

   ```python
   """Self-contained HTML report generation."""

   from __future__ import annotations

   import json
   from datetime import datetime, timezone
   from pathlib import Path

   from jinja2 import Environment, PackageLoader

   from open_pacmuci.version import __version__


   def generate_report(
       summary: dict,
       output_path: Path,
       sample_name: str = "unknown",
       tool_versions: dict[str, str] | None = None,
   ) -> Path:
       """Generate a self-contained HTML report from pipeline results.

       Args:
           summary: Combined pipeline summary (alleles + classification).
           output_path: Where to write the HTML file.
           sample_name: Sample identifier for the report header.
           tool_versions: Optional dict of tool name -> version string.

       Returns:
           Path to the generated HTML report.
       """
       env = Environment(
           loader=PackageLoader("open_pacmuci", "templates"),
           autoescape=True,
       )
       template = env.get_template("report.html.j2")

       html = template.render(
           sample_name=sample_name,
           summary=summary,
           tool_versions=tool_versions or {},
           pipeline_version=__version__,
           generated_at=datetime.now(timezone.utc).isoformat(),
       )

       output_path.parent.mkdir(parents=True, exist_ok=True)
       output_path.write_text(html)
       return output_path
   ```
